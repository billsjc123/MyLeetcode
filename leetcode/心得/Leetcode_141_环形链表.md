# 环形链表1-判断是否有环

时间复杂度肯定是O(N)，关键在于空间复杂度如何从O(N)优化为O(1)

## Hash
正常思路肯定是保存每一个ListNode并进行判定，用HashSet作为数据结构存储经过的结点是最快的。
## Floyd算法
想优化为O(1)则需要该思想：如果有环，并且两个指针前进速度不一样，那么一定能相遇。因此每次循环一个指针走两步一个指针走一步即可。

# 环形链表2-找到环开始的地方

同样可以不使用额外内存空间

## Floyd算法
判断是否有环中，使用了快慢指针来相互追逐判断是否有环。让我们假设head到环头的距离为F，快慢指针第一次相遇的地方到环头的距离是a，从相遇的地方回到环头的距离是b（a+b=环长），那么我们可以列出如下公式：
$$2(F+a) = F + a + b + a$$
$$F = b$$
因此从相遇点再次回到环头的距离正好就是从head到环头的距离。
因此我们让指针指向环头，和慢指针以相同速率前进，相遇点即为环头。
